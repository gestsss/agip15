# Документация проекта

Данный репозиторий содержит проектную документацию,
описывающую планирование, объём работ, критерии готовности
и риски.


## Содержание

- [План выполнения работ](docs/plan.md)
- [Объём работ](docs/scope.md)
- [Критерии готовности и приёмка](docs/dod-and-acceptance.md)
- [Риски проекта](docs/risks.md)

---


- работа организована в два спринта;
- используется минимально достаточный объём функциональности;
- учитываются ограничения по времени участников;
- основной приоритет - сквозной пользовательский сценарий;
- в оценки включены синки и коммуникации.

## Примечание

В рамках общего лимита проекта я предусмотрела резерв времени
на возможные риски и задачи.

Резерв не выделяется отдельной строкой, а распределён внутри плана,
в первую очередь за счёт увеличенного объёма работ по тестированию
и стабилизации во втором спринте.
