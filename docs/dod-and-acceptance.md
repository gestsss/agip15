# Критерии готовности и приёмка результата

## Назначение документа

Данный документ описывает критерии готовности функциональности
и порядок приёмки результата работ в рамках проекта.

---

## Definition of Done

Функциональность считается готовой, если выполнены все условия ниже.

### Функциональные критерии
- реализован основной пользовательский сценарий;
- функциональность соответствует согласованным требованиям;
- данные корректно передаются и обрабатываются на клиенте и сервере;
- ошибки обрабатываются предсказуемо и не блокируют основной сценарий.

### Критерии качества
- отсутствуют блокирующие и критические дефекты;
- устранены дефекты, влияющие на основной пользовательский сценарий;
- проведена проверка ключевых пользовательских потоков.

### Технические критерии
- изменения добавлены в основную ветку репозитория;
- сборка проекта проходит успешно;
- приложение задеплоено и доступно в тестовом окружении;
- базовые проверки CI выполняются без ошибок.

### Процессные критерии
- задача проверена специалистом по качеству;
- результат принят менеджером проекта;
- статус задачи зафиксирован как завершённый.

---

## Процесс приёмки результата

1. Исполнитель завершает реализацию задачи.
2. Результат передаётся на проверку качества.
3. QA проводит проверку по чек-листу и тест-кейсам.
4. При обнаружении дефектов задача возвращается на доработку.
5. После исправлений проводится повторная проверка.
6. PM принимает результат по критериям готовности.
7. Задача переводится в статус завершённой.

---

## Приёмка спринта

Спринт считается завершённым, если:
- достигнута цель спринта;
- основной пользовательский сценарий работает стабильно;
- отсутствуют блокирующие дефекты;
- проведена демонстрация результата.
